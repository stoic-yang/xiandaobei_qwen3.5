# 40 · 用户背景与协作偏好

## 用户自述
对 vLLM 等推理系统比赛相关知识"停留在非常浅显的理解"，并主动要求：交流中遇到模糊之处时帮他完善、填充并明确指出。

**How to apply**：解释方案时附带最小必要的概念铺垫（prefill/decode、TTFT/TPOT、KV cache、带宽受限等），指出用户表述中含糊或有误的地方并给出修正，而不是默认已理解。

## 协作模式
- 这是队伍作战（非单人）。队友已经搭好远超"环境搭建"的基础设施，用户本人此前并不知情。
- 涉及"环境处于什么状态"，先跑诊断命令确认现状再行动，不假设"从零开始"或"队友做的都在"。
- 发现有价值的既有工作时，优先建议用户直接找队友同步，而不是继续靠文件考古。

## 工具链偏好
- 同时使用 **Codex / Claude Code / OpenCode** 三个 agent —— 因此需要项目内共享 meta 仓库（见 `AGENTS.md`），而非任一 agent 的私有记忆。
- SSH 三段跳打通后，agent 可远程直连容器，无需用户网页终端人肉中转。
